  1. Is the receipt PDF generation service currently down or experiencing issues?
 
  No. QuestPDF is not the problem — the error happens before QuestPDF is even reached. The exception occurs during the SP call (POS_MultiReceiptPrint_sp) or the

  2. Does this affect all receipts or only new ones created today?
 
  It depends on the root cause. Two scenarios:
 
  - If Bug B is the cause (duplicate receipt numbers in GetReceiptIdsByNosAsync): only receipts whose receipt number matches a row from a prior financial year are
  affected. Receipt numbers follow the pattern ddMMyyyy/sequence. Any cashier that had sequence number 460027 on March 6 of a prior year would collide with today's
   06032026/460027. This would be intermittent — some receipts print fine, others don't, depending on whether the number existed before.
  - If the SP itself is failing (e.g. schema change, missing object): all receipts would be affected regardless of date.
 
  The fix to GetReceiptIdsByNosAsync is now deployed. After that fix, call print-receipt again. If it now returns a meaningful error message in the 500 body
  (instead of the generic problem details), it means Bug B was NOT the issue and the SP itself is failing — the message will say exactly why.
