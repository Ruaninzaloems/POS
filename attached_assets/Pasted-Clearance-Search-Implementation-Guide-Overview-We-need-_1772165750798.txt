Clearance Search Implementation Guide
Overview
We need to implement a two-step search flow for clearance records:
Step 1: Search/Typeahead - GET Request
Endpoint: GET api/billing-payment-clearance/get-clearanceids
Purpose: Auto-suggest/display matching clearance IDs as user types
Query Parameter: clearanceId - The partial ID entered by user (e.g., "000000001612")
Example Request:
text
 
Plain Text
GET api/billing-payment-clearance/get-clearanceids?clearanceId=160
 
Example Response:
json
 
Plain Text
[  "000000000160",  "000000001160"]
 
Behavior:
Returns list of matching formatted IDs (12 digits with leading zeros)
User selects one ID from this list
Step 2: Fetch Full Data - POST Request
Endpoint: POST api/billing-payment-clearance/get-clearance-data
Purpose: Retrieve complete clearance details after user selects an ID
Request Body: Raw integer ID (without formatting)
json
 
Plain Text
"000000000160"
